# Contributing to Nim → Amiga 500

Thank you for wanting to help! 🙌

## 🧭 No coding experience? No problem.

You can contribute by:

- ✍️ **Improving documentation**: Fix typos, clarify steps, add examples
- 🧪 **Testing**: Run the examples on your setup, report what works or breaks
- 💡 **Suggesting ideas**: What program would you like to see for Amiga?
- 🌍 **Translating**: Help make docs available in your language
- 📣 **Sharing**: Tell other Amiga or Nim fans about this project

## 💻 Know how to code?

Great! Here's where to start:

### Good first issues
Look for issues labeled [`good first issue`](https://github.com/mcphisto-netizen/nim-amiga/issues?q=label%3A%22good+first+issue%22)

### Areas that need help
- 📚 More example programs (`examples/` folder)
- 🔧 Better error messages in build scripts
- 🎨 Visual improvements (screenshots, diagrams)
- 🧪 Automated testing helpers

### Code guidelines
- Keep runtime size in mind (<10 KB target)
- Target Motorola 68000 only (no 68020+ instructions)
- Avoid libc dependencies
- Comment non-obvious m68k assembly

## 🗣️ Questions or ideas?

- Open a [Discussion](https://github.com/mcphisto-netizen/nim-amiga/discussions)
- Join the conversation in existing issues
- Be kind — we're all learning together ❤️

## 🔄 How to submit changes

1. Fork the repo (click "Fork" at the top right)
2. Create a branch: `git checkout -b my-feature`
3. Make your changes
4. Test if possible
5. Commit: `git commit -m "Add: my helpful change"`
6. Push: `git push origin my-feature`
7. Open a Pull Request on GitHub

We'll review and help you get it merged!

## 📜 Code of Conduct

Be respectful, patient, and constructive. This is a passion project — let's keep it fun for everyone.
